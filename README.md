# CAPTCHAVision: Breaking Text CAPTCHAs with CNNs

> A two-part ML project targeting text-based and image-based CAPTCHAs.  
> Team: Salome Shioshvili · Jinin Aljayyousi · Nikita Djandjgava · Dana Dabbah

---

## What this does

Takes a 4-character CAPTCHA image like this and automatically predicts the answer (Part 1), and identifies matching tiles in a Google reCAPTCHA v2 image grid (Part 2).

The Part 1 target is the **Really Simple CAPTCHA** WordPress plugin with over 1 million active installations. The plugin's author himself says it's no longer secure, which makes it a good study target without ethical concerns.

## How to run

### Part 1 — Text CAPTCHA

#### Step 1 — Extract individual letters from CAPTCHAs

```bash
cd part1
python extract_letters.py
```

Reads every image in `generated_captcha_images/`, segments it into 4 letter crops using OpenCV contour detection, and saves them to `extracted_letter_images/<LETTER>/`. Images where segmentation fails (e.g. too many merged letters) are skipped automatically.

#### Step 2 — Train the CNN

```bash
python train_model.py
```

Loads all extracted letter images, trains a CNN on them, and saves the model. Takes a few minutes on CPU.

#### Step 3 — Solve CAPTCHAs

```bash
# evaluate on a folder (filename must be the correct answer)
python solve_captchas.py

# solve a single image
python solve_captchas.py path/to/captcha.png
```

### Part 2 — Image CAPTCHA (reCAPTCHA v2)

#### Step 1 — Train the model

```bash
cd part2
python train_model.py
```

Downloads the Kaggle dataset automatically via `kagglehub`, trains MobileNetV2 with two-phase fine-tuning, and saves the model.

#### Step 2 — Solve a grid

```bash
python solve_grid.py path/to/grid.png <category> [grid_size]
# example:
python solve_grid.py captcha.png traffic_light 3
```

## How the segmentation works (Part 1)

Raw CAPTCHA → grayscale → binary threshold (Otsu) → find contours → sort left to right → save each crop.

The only tricky part: sometimes two letters overlap and OpenCV sees them as a single blob. We handle this by checking if a region is much wider than it is tall — if it is, we split it down the middle and treat each half as a separate letter.

## Model architectures

### Part 1 — Baseline CNN (Geitgey 2017)

```
Input (20×20×1)
→ Conv2D(20, 5×5) + ReLU + MaxPool
→ Conv2D(50, 5×5) + ReLU + MaxPool
→ Dense(500) + ReLU
→ Dense(32) + Softmax
```

### Part 2 — MobileNetV2 + fine-tuning

```
Input (96×96×3)
→ MobileNetV2 base (pretrained on ImageNet, frozen in Phase 1)
→ GlobalAveragePooling2D
→ Dense(128) + ReLU
→ Dropout(0.3)
→ Dense(12) + Softmax
```

Training proceeds in two phases: 5 epochs with the base frozen, then 5 more epochs fine-tuning the last 20 layers at lr=1e-5.

## Datasets

**Part 1:** 10,000 CAPTCHA images generated from the WordPress plugin source code. ~40,000 individual character images after segmentation (~1,000–1,200 per class). No manual labelling — the filename encodes the answer.

**Part 2:** Google reCAPTCHA V2 Images Dataset (Kaggle: mikhailma/test-dataset). 11,730 images across 12 categories. Significant class imbalance (Car: 3,558 images vs Motorcycle: 81). Class weights applied during training to compensate.

## Limitations

- Part 1 is trained only on Really Simple CAPTCHA images — will not generalise to other styles without retraining
- Part 1 segmentation assumes exactly 4 characters; will fail on variable-length CAPTCHAs
- The split-in-half fix for merged letters is a heuristic and occasionally wrong
- Part 2 struggles with underrepresented classes (Motorcycle F1=0.48, Chimney F1=0.47)
- Part 2 may not generalise to live reCAPTCHA grids which are continuously updated by Google
